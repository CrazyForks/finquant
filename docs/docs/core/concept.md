# 核心概念

FinQuant 采用事件驱动架构，主要包含以下核心概念：

## 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                     FinQuant 架构                        │
└─────────────────────────────────────────────────────────┘

┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 数据源   │───▶│ 事件总线  │───▶│  策略    │───▶│  风控    │
│ Data    │    │ EventBus │    │ Strategy │    │ RiskMgr  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                      │                 │                 │
                      ▼                 ▼                 ▼
                 ┌──────────────────────────────────────────┐
                 │              回测引擎 Engine             │
                 └──────────────────────────────────────────┘
```

## 核心组件

### 1. 数据源 (Data)

数据源负责获取和处理市场数据：

- 本地 Parquet 缓存
- 多数据源自动切换
- 增量更新机制

### 2. 事件总线 (Event Bus)

事件总线是框架的核心，负责协调各组件：

- `on_bar` - K线数据事件
- `on_order` - 订单事件
- `on_trade` - 成交事件
- `on_signal` - 信号事件

### 3. 策略 (Strategy)

策略负责生成交易信号：

```python
class MyStrategy(BaseStrategy):
    def on_bar(self, bar):
        # 生成买入/卖出信号
        if self.condition:
            self.buy(bar.close, quantity)
```

### 4. 风控 (Risk Manager)

风控模块提供：

- 止损/止盈
- 最大回撤控制
- 仓位管理
- 交易限制

## 关键设计原则

1. **解耦** - 策略与引擎完全解耦
2. **事件驱动** - 通过事件总线通信
3. **可扩展** - 组件可插拔
4. **精度优先** - 真实模拟交易场景
